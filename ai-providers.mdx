AI Puffer uses provider settings for chat, writing, automations, AI forms, images, speech, embeddings, and knowledge features.

Set up the providers you want to use, sync their models, then choose the provider and model inside each module.

> Screenshot placeholder: Settings screen with the AI tab selected.

## Provider areas

AI Puffer has two provider areas in **AI Puffer > Settings**.

| Area | Use it for |
| --- | --- |
| **AI** | Default AI provider, default model, API key, model sync, and advanced connection settings. |
| **Integrations** | Service credentials for ElevenLabs, Replicate, Pinecone, Qdrant, Pexels, and Pixabay. |

Settings save automatically after you change a field.

## Supported providers

| Feature area | Providers |
| --- | --- |
| Chat, Content Writer, Automations, AI Forms | OpenAI, Google, Claude, OpenRouter, Azure, DeepSeek, Ollama |
| Embeddings | OpenAI, Google, Azure, OpenRouter, Ollama |
| Vector stores | OpenAI Vector Stores, Pinecone, Qdrant |
| Image generation | OpenAI, Google, Azure, OpenRouter, Replicate |
| Stock image search | Pexels, Pixabay |
| Video generation | Google video models |
| Text to speech | Google, OpenAI, ElevenLabs |
| Speech to text | OpenAI, Azure |
| Web search / grounding | OpenAI, Google, Claude, OpenRouter |
| Realtime voice | OpenAI realtime models |

Ollama requires the Pro plan in the plugin UI.

## Main AI providers

Use the **AI** tab for these providers.

| Provider | Required setup | Notes |
| --- | --- | --- |
| OpenAI | API key | Supports chat models, embeddings, image generation, speech, realtime voice, moderation, and OpenAI vector stores. |
| Google | API key | Supports Gemini models, embeddings, image models, video models, text to speech, and Google grounding. |
| Claude | API key | Supports chat and writing workflows. Web search and image analysis are available where the selected Claude model supports them. |
| OpenRouter | API key | Routes to models available through OpenRouter. Image, embedding, and web search support depends on the selected model or route. |
| Azure | API key and endpoint URL | Uses Azure OpenAI deployments. Sync deployments before selecting a model. |
| DeepSeek | API key | Supports chat and writing workflows. Dedicated embeddings are not supported by this provider strategy. |
| Ollama | Base URL | Uses a local or self-hosted Ollama server. No API key is required. The WordPress server must be able to reach the Ollama URL. |

## Service integrations

Use the **Integrations** tab for these services.

| Integration | Used for | Required setup |
| --- | --- | --- |
| ElevenLabs | Text to speech voices and models | API key, then sync voices and models. |
| Replicate | Image generation models | API key, then sync models. |
| Pinecone | Vector database indexes | API key, then sync indexes. |
| Qdrant | Vector database collections | Qdrant URL and API key, then sync collections. |
| Pexels | Stock image search | API key. |
| Pixabay | Stock image search | API key. |

Pexels and Pixabay search existing stock images. They do not generate AI images.

## Set the default AI provider

1. Open **WordPress Admin > AI Puffer > Settings**.
2. Select the **AI** tab.
3. Choose a provider from **Provider**.
4. Enter the required credential:
   - For OpenAI, Google, Claude, OpenRouter, Azure, or DeepSeek, enter the API key.
   - For Azure, also enter the endpoint URL.
   - For Ollama, enter the Ollama base URL.
5. Open **Advanced**.
6. Click **Sync Models**.
7. Choose the default model.

## Sync all providers

Use this when you changed several API keys or want to refresh all model lists.

1. Open **AI Puffer > Settings**.
2. Select the **Others** tab.
3. Click **Sync All**.
4. Wait for the sync to complete.

AI Puffer only syncs providers that have the required credentials or connection settings.

> Screenshot placeholder: Others tab with the Sync All button.

## OpenAI

Use OpenAI when you want OpenAI models for chat, writing, forms, automations, images, embeddings, vector stores, speech, or realtime voice.

1. Open **AI Puffer > Settings > AI**.
2. Select **OpenAI**.
3. Paste your OpenAI API key.
4. Keep the default base URL unless you use a compatible custom endpoint.
5. Open **Advanced** and click **Sync Models**.
6. Select the model you want to use by default.

For knowledge features, OpenAI embedding models are available in embedding model selectors. OpenAI Vector Stores are handled in Knowledge Base.

## Google

Use Google for Gemini text models, Google embedding models, image models, video models, text to speech, and grounding.

2. Select **Google**.
3. Paste your Google API key.
4. Open **Advanced** and click **Sync Models**.
5. Select the default model.

## OpenRouter

Use OpenRouter when you want to route through models available in your OpenRouter account.

1. Open **AI Puffer > Settings > AI**.
2. Select **OpenRouter**.
3. Paste your OpenRouter API key.
4. Open **Advanced** and click **Sync Models**.
5. Select the default model.

OpenRouter model capabilities vary. Image generation, image editing, embeddings, and web search require a model or route that supports that capability.

## Azure

Use Azure when your site should use Azure OpenAI deployments.

1. Open **AI Puffer > Settings > AI**.
2. Select **Azure**.
3. Paste your Azure API key.
4. Enter the Azure endpoint URL for your resource.
5. Open **Advanced**.
6. Check the API versions if your Azure resource requires different versions.
7. Click **Sync Models** to load deployments.
8. Select the deployment to use as the default model.

Azure uses deployment names. If a deployment does not appear, confirm it exists in Azure and that the endpoint and API key belong to the same resource.

## Ollama

Use Ollama for local or self-hosted models.

1. Install and run Ollama on a server that WordPress can reach.
2. Pull the model you want to use in Ollama.
3. Open **AI Puffer > Settings > AI**.
4. Select **Ollama**.
5. Enter the Ollama base URL. The default is `http://localhost:11434`.
6. Click **Sync Models**.
7. Select the default model.

If WordPress and Ollama are on different servers, do not use `localhost` unless Ollama is running on the same server as WordPress. Use the reachable server URL instead.

## Use a provider in a module

After provider setup, choose the provider and model in the module you are configuring.

1. Open the module, such as **Chat**, **Content Writer**, **Automations**, or **AI Forms**.
2. Select the provider.
3. Select the model.
4. Configure module-specific options.
5. Save or publish the module item.

Modules can use a different provider than the global default.

## Troubleshooting

### The model list is empty

Check that the API key or base URL is saved, then sync models again.

For Azure, confirm the endpoint URL and deployments. For Ollama, confirm WordPress can reach the Ollama base URL.

### A module says the provider is not configured

Open **AI Puffer > Settings** and confirm the provider has the required API key, endpoint, base URL, or synced models.

### Embeddings do not work with the selected provider

Use one of the embedding providers: OpenAI, Google, Azure, OpenRouter, or Ollama.

### Image generation does not work with OpenRouter

Select an OpenRouter model that supports image output. For image editing, the model must support both image input and image output.

### Speech settings do not show the provider you expected

Text to speech supports Google, OpenAI, and ElevenLabs. Speech to text supports OpenAI and Azure.

### Ollama works locally but not from WordPress

The URL is resolved from the WordPress server, not from your browser. If Ollama runs on your computer and WordPress runs on another server, `http://localhost:11434` will not point to your computer.
